# TravelLine-CMS: Система управления контентом корпоративного портала

Легковесная и отказоустойчивая динамическая CMS, разработанная в рамках учебной практики. Проект представляет собой административную панель для управления контентом и модулями веб-платформы на примере интерфейса для ИТ-компании **TravelLine Tech**.

## 🚀 Особенности и функционал

* **Файловая БД (JSON):** Хранение всей структуры данных в едином дереве `data.json` без использования тяжеловесных СУБД, что обеспечивает максимальное быстродействие.
* **Управление блоками (CRUD):** * Редактирование главного экрана (Hero-блок).
    * Динамическое добавление и удаление карточек сотрудников («Команда»).
    * Управление списком вакансий и направлениями разработки.
    * Полный цикл управления корпоративными бенефитами («Плюшки и все такое»).
* **Безопасность:** Защита от XSS-атак с помощью принудительного экранирования вывода через `htmlspecialchars()`, валидация серверных POST-запросов и защита от повторной отправки форм (Double Submit F5) через 302-редиректы.
* **Современный UI:** Интерфейс административной панели выполнен в концепции *Glassmorphism* (эффект матового стекла, парящие элементы, размытие заднего плана).

## 🛠 Стек технологий

* **Backend:** PHP 8+ (Контроллер обработки мутаций данных, switch-case маршрутизация)
* **Database:** JSON (Древовидная структура ассоциативных массивов)
* **Frontend:** HTML, CSS (Переменные `:root`, Glassmorphism CSS, Flexbox/Grid), JavaScript (GSAP анимации)
* **Стандарты:** Соблюдение рекомендаций PSR (стандарты кодирования)

## 📐 Архитектура данных

Данные в `data.json` хранятся в виде структурированного ассоциативного массива. При удалении элементов производится обязательная переиндексация ключей через `array_values()`, что предотвращает деградацию JSON-массива в объект.

Математический алгоритм распределения CSS-классов на фронтенде:
$$ItemNumber = (Index \pmod 8) + 1$$

Это гарантирует равномерное закольцовывание стилей от `1` до `8` при любом объеме динамического контента.
